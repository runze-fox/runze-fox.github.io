# Runze Shi Portfolio Site

This is a static personal portfolio website for project-based job applications. It uses only HTML, CSS, and a small amount of native browser behavior, so it is suitable for GitHub Pages.

## How to edit project content

- Edit the homepage content in `index.html`.
- Edit project detail pages in the `projects/` folder:
  - `projects/hubble.html`
  - `projects/welding.html`
  - `projects/satellite.html`
  - `projects/dji.html`
- Each project page follows the same structure:
  - Project Title
  - Project Summary
  - Motivation / Background
  - Key Results
  - Selected Figures
  - My Contribution
  - Code / Report
  - Back to Portfolio

## How to replace images

Replace the files in the `images/` folder while keeping the same filenames:

- `profile-placeholder.jpg`
- `hubble-cover.png`
- `welding-cover.png`
- `satellite-cover.png`
- `dji-cover.png`

Recommended image sizes:

- Project covers: 1600 x 900 px or any 16:9 image.
- Profile image: square image, such as 800 x 800 px.

## How to deploy on GitHub Pages

1. Create a new GitHub repository.
2. Upload all files in this folder to the repository.
3. Go to the repository settings.
4. Open **Pages**.
5. Set the source branch to `main`.
6. Save the settings.
7. GitHub will publish the site after the Pages build finishes.

## Recommended repository name

For a personal GitHub Pages website, use:

```text
<github-username>.github.io
```

If your repository has this exact name, GitHub Pages can serve it as your main personal website.
